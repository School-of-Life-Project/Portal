î‡ Unlike importing existing resources, creating resources for Portal is a more involved progress. A degree of specialized technical knowledge will be required.

You will need a basic understanding of the following concepts:

  • Tom's Obvious Minimal Language (TOML)
  • Universally Unique Identifier (UUID)
  • JSON Schema
  • EPUB e-Book format
    • EPUB v3.2 specification
    • Open Publication Structure v2 specification (only used by EPUB v2 documents)
    • Extensible Markup Language (XML)
    • Hypertext Markup Language (HTML)
    • OCF abstract container (EPUB-specific file hierarchy)
    • EPUB navigation document
    • Navigation Center eXtended (only used by EPUB v2 documents)
  • ZIP archive format
    • DEFLATE compression algorithm
    • LZMA compression algorithm

You will need the following digital tools:

  • A file archiver, such as PeaZip
  • A code editor, such as Visual Studio Code
    • A plugin to validate TOML files against a JSON schema (recommended)
  • A specialized ePub editor, such as Sigil (recommended)

Throughout the rest of this section, Portal's user data folder will be referred to as $APPDATA.

î‡ Portal Schema

Portal preserves backwards-compatibility whenever possible. Updating Portal should not break existing resources â€” if this happens, it's a bug.

However, Portal makes no guarantees regarding forwards-compatibility for resources. Resources designed for newer versions may fail to load on older versions.

As a result, it may be preferable to use specific versions of Portal for building resources. Portal automatically generates a folder containing JSON schema upon startup, located at $APPDATA/Resource Schema/. This schema documents the structure and functionality of resource TOML files.

This manual aims to maintain a degree of both backwards and forwards compatibility, and will not repeat information that is within the scope of the relevant schema files.
